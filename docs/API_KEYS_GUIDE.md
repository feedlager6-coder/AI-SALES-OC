## Карта интеграций

```
Поток данных:
                     ┌─────────────┐
                     │  2ГИС API   │ ← ищем компании по отрасли + городу
                     │  HH.ru API  │ ← сигналы о росте (вакансии)
                     └──────┬──────┘
                            │ сырые компании
                     ┌──────▼──────┐
                     │  Dadata.ru  │ ← ИНН, ОГРН, статус, руководитель
                     │  ЕГРЮЛ      │ ← бесплатный fallback
                     └──────┬──────┘
                            │ обогащённые компании
                     ┌──────▼──────┐
                     │  Hunter.io  │ ← email по домену (приоритет 1)
                     │  Snov.io    │ ← email по домену (приоритет 2)
                     │  Pattern    │ ← guess email (fallback, без API)
                     └──────┬──────┘
                            │ email найден
                     ┌──────▼──────┐
                     │   OpenAI    │ ← персонализированный текст письма
                     │  Anthropic  │ ← fallback если OpenAI недоступен
                     └──────┬──────┘
                            │ готовое письмо
                     ┌──────▼──────┐
                     │   Mailgun   │ ← отправка + open/click tracking
                     │   Brevo     │ ← резервный провайдер
                     └──────┬──────┘
                            │ ответ пришёл
                     ┌──────▼──────┐
                     │   OpenAI    │ ← классификация: интерес / отказ / вне офиса
                     └──────┬──────┘
                            │
                     ┌──────▼──────┐
                     │  Telegram   │ ← алерт SDR "лид ответил, хочет звонок"
                     └─────────────┘
```

**Почему именно такая цепочка?** Каждый слой отвечает за одну задачу и заменяем независимо. Если Hunter поднимет цены — меняем на Snov без правок в воркерах. Если OpenAI упадёт — Anthropic подхватит автоматически. Mailgun → Brevo — то же самое.

## P1 — Обязательные

### 1. OpenAI — Мозг системы

**Зачем:** Генерирует текст каждого исходящего письма на основе данных о компании. Классифицирует входящие ответы (интерес / отказ / OOO / вопрос). Без него шаблонные письма — единственная альтернатива.

**Используемые модели в коде:**
- `gpt-4o-mini` — генерация писем (дёшево, достаточно умно)
- `gpt-4o` — классификация ответов (точнее, разумная цена)

**Бесплатный план:** ❌ Нет. Только $5 trial-кредит при регистрации (хватит на ~3–5 тыс. писем с gpt-4o-mini).

**Стоимость (gpt-4o-mini):**
| Действие | Токены | Стоимость |
|----------|--------|-----------|
| Генерация 1 письма (контекст + промпт + ответ) | ~1,500 | ~$0.0003 |
| 1,000 писем/день | ~1.5M | ~$0.30/день |
| Классификация 1 ответа | ~500 | ~$0.0001 |

**Почему OpenAI, а не другой LLM:**
- Лучшее соотношение русский язык / цена / API стабильность
- gpt-4o-mini обучен на русскоязычных текстах значительно лучше, чем Mistral/Llama
- Function calling для классификации работает надёжнее других провайдеров
- Низкая латентность (<2 сек на письмо) критична для UX воркера

**Альтернативы:**
| Провайдер | Плюсы | Минусы | Вывод |
|-----------|-------|--------|-------|
| Anthropic Claude 3.5 Haiku | Хороший русский, дешевле Sonnet | Дороже gpt-4o-mini; нет европейского датацентра | Хороший fallback, не замена |
| Mistral (mistral-small) | Есть EU датацентр | Слабее на русском языке B2B копирайтинге | P3 |
| YandexGPT | Лучший русский | Нет function calling; закрытый API; нет webhooks | Не подходит для классификации |
| GigaChat | Российский датацентр | API нестабильный; нет структурированного вывода | P3 |
| Gemini 1.5 Flash | Дешевле gpt-4o-mini | Хуже работает с B2B промптами на русском | P3 |

**Как получить (пошагово):**
1. Перейти на **[platform.openai.com](https://platform.openai.com)**
2. Нажать **"Sign up"** → зарегистрироваться (можно Google-аккаунт)
3. Подтвердить телефон (обязательно, без него API недоступен)
4. В левом меню: **"API Keys"** → **"Create new secret key"**
5. Дать имя ключу: `ai-sales-os-production`
6. Скопировать ключ — он показывается **один раз**
7. В левом меню: **"Billing"** → **"Add payment method"** → добавить карту
8. Установить **Usage limit** (рекомендуем $20/мес на старте): **"Billing" → "Limits"**

> ⚠️ Без добавления карты API работает только на trial $5. После исчерпания — 429 ошибки.

**Env var:** `OPENAI_API_KEY=sk-proj-...`

### 2. Mailgun — Email отправка

**Зачем:** Единственный способ отправить письмо и знать, что с ним произошло. Mailgun даёт webhooks на каждое событие (доставлено, открыто, кликнуто, bounce, spam complaint, unsubscribe). Без него нет статистики кампании.

**Бесплатный план:** ✅ **100 писем/день, 3 месяца** (Flex Trial). Достаточно для MVP и первых пилотов.

**Платные тарифы:**
| Тариф | Цена | Лимит | Хватит для |
|-------|------|-------|-----------|
| Foundation | $35/мес | 50,000 писем | ~1,600 писем/день |
| Scale | $90/мес | 100,000 писем | ~3,300 писем/день |

**Почему Mailgun:**
- Лучший deliverability среди transactional email провайдеров (Litmus 2025: 98.7% inbox rate)
- Единственный из топ-провайдеров с полноценным REST API для tracking без дополнительных инструментов
- Sandbox domain работает из коробки (не нужен реальный домен для старта)
- Webhooks подписываются на конкретный домен, не на весь аккаунт — удобно при мультитенантности

**Альтернативы:**
| Провайдер | Бесплатно | Цена | Deliverability | Вывод |
|-----------|-----------|------|----------------|-------|
| **Brevo** | **300/день навсегда** | €25/мес за 20K | Хорошая | Лучший бесплатный тир — резервный P2 |
| SendGrid | 100/день навсегда | $19.95/мес | Отличная | Дороже; UI хуже |
| Postmark | Нет | $15/мес за 10K | Лучшая | Нет open tracking на бесплатном |
| Amazon SES | Нет | $0.10 за 1K | Хорошая | Нужна настройка IAM; нет webhooks из коробки |
| SMTP Яндекс | Нет | Бесплатно (Яндекс 360) | Плохая для cold | Не для cold email |

**Как получить (пошагово):**

**Шаг 1 — Регистрация:**
1. Перейти на **[app.mailgun.com/signup](https://app.mailgun.com/signup)**
2. Заполнить форму → подтвердить email
3. Ввести номер телефона для верификации

**Шаг 2 — Получить API ключ:**
1. Войти → слева меню **"API Security"** (или Settings → API Security)
2. Нажать **"Add new key"** → выбрать тип **"Sending"**
3. Скопировать ключ (начинается с `key-...`)

**Шаг 3 — Sandbox домен (для старта, без DNS):**
1. В левом меню: **"Sending" → "Domains"**
2. Уже есть домен `sandbox-XXXX.mailgun.org` — использовать его
3. Нажать на домен → **"Authorized Recipients"** → добавить свой email для тестов
4. Значение `MAILGUN_DOMAIN` = `sandbox-XXXX.mailgun.org`

**Шаг 4 — Реальный домен (для продакшна):**
1. **"Sending" → "Domains" → "Add new domain"**
2. Вводить поддомен, например `mail.yourdomain.ru` (не root домен!)
3. Mailgun покажет DNS записи — добавить в панели регистратора:
   - 2x TXT записи (SPF и DKIM)
   - 1x CNAME для tracking
4. Нажать **"Verify DNS settings"** — обычно 5–30 минут

**Шаг 5 — Webhooks для tracking:**
1. **"Sending" → "Webhooks"**
2. Выбрать домен → **"Add webhook"**
3. URL: `https://your-domain.replit.dev/api/webhooks/mailgun`
4. Отметить все события: `delivered`, `opened`, `clicked`, `bounced`, `complained`, `unsubscribed`

**Env vars:**
```
MAILGUN_API_KEY=key-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
MAILGUN_DOMAIN=sandbox-XXXX.mailgun.org   # или mail.yourdomain.ru
```

### 3. Dadata.ru — Российский реестр компаний

**Зачем:** Единственный способ получить структурированные данные из ФНС о российских компаниях: ИНН, ОГРН, ФИО руководителя, статус (действующая/ликвидирована), ОКВЭД, адрес ФИАС. Без этого обогащение компаний из 2ГИС неполное.

**Бесплатный план:** ✅ **10,000 запросов/сутки** — хватит на ~10K компаний/день. Для MVP более чем достаточно.

**Платный:** ₽0.50–2.00 за запрос при превышении. При 50K компаний/день = ~₽50K/мес — заранее договориться о корпоративном тарифе.

**Почему Dadata, а не прямой ЕГРЮЛ:**
- ЕГРЮЛ не имеет официального JSON API (только HTML + PDF)
- Dadata кэширует и нормализует данные — ответ за 50–200ms vs 2–5 сек у парсера ЕГРЮЛ
- Стандартизация адресов (ФИАС) уже встроена — не нужен отдельный геокодер
- API поддерживает fuzzy-search по названию (ЕГРЮЛ требует точное ИНН)

**Альтернативы:**
| Сервис | Стоимость | Качество | Вывод |
|--------|-----------|----------|-------|
| **Dadata** | Free 10K/day | ⭐⭐⭐⭐⭐ | Выбор №1 |
| Контур.Фокус | ~₽30K/мес | ⭐⭐⭐⭐⭐ | Лучше данных, нет Dev API |
| СБИС API | Договорной | ⭐⭐⭐⭐ | Нет публичного тарифа |
| ФНС OpenData | Бесплатно | ⭐⭐⭐ | Только bulk XML, нет real-time API |
| Rusprofile парсинг | Бесплатно | ⭐⭐ | Нарушение ToS + нестабильно |

**Как получить (пошагово):**
1. Перейти на **[dadata.ru](https://dadata.ru)**
2. Нажать **"Подключиться"** (правый верхний угол)
3. Зарегистрироваться: email + пароль (или Яндекс OAuth)
4. Подтвердить email
5. В личном кабинете → **"API-ключи"** (левое меню)
6. Будет один ключ по умолчанию — скопировать **"Токен"**

> API-ключ и Secret token — это разные вещи. В нашем коде используется только `DADATA_API_KEY` (токен). Secret нужен только для очищения данных (не наш кейс).

**Env var:** `DADATA_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

### 4. 2ГИС — Поиск лидов в РФ

**Зачем:** Основной источник лидов. 2ГИС — крупнейшая база организаций в России и СНГ (~5 млн компаний с телефонами, адресами, категориями). API позволяет искать по отрасли + городу + радиусу — идеально для territory-based продаж.

**Бесплатный план:** ✅ **Demo ключ** — 1,000 запросов/день, без доступа к Catalog API. Достаточно для разработки.

**Платный:** от ₽5,000/мес — полный доступ к Catalog API (поиск по категории, все атрибуты).

> ⚠️ Demo ключ даётся автоматически при регистрации и работает для Maps API. **Catalog API** (поиск организаций по рубрике) требует платного тарифа или явного подтверждения от 2ГИС.

**Почему 2ГИС, а не Яндекс Бизнес / Google Places:**
- Единственный провайдер с полной базой по РФ/СНГ + правильными рубриками ОКВЭД
- Google Places API: нет API для массового поиска по отрасли; дорого ($17 за 1K detail requests)
- Яндекс Бизнес: нет публичного API для поиска по категории для сторонних разработчиков
- 2ГИС — официальный API, можно перепродавать аналитику (нельзя только сырые данные)

**Альтернативы:**
| Сервис | Покрытие РФ | API | Цена | Вывод |
|--------|-------------|-----|------|-------|
| **2ГИС** | ⭐⭐⭐⭐⭐ | REST | ₽5K+/мес | Выбор №1 для РФ |
| Яндекс Бизнес | ⭐⭐⭐⭐ | Нет публичного | N/A | Нет API |
| Google Places | ⭐⭐⭐ | REST | ~$17/1K | Нет отраслевого поиска |
| HH.ru | ⭐⭐⭐⭐ | REST | Бесплатно | Не компании, а работодатели |
| OpenStreetMap/Overpass | ⭐⭐ | Сложный | Бесплатно | Плохое покрытие бизнесов |

**Как получить (пошагово):**

**Demo ключ (для разработки):**
1. Перейти на **[dev.2gis.ru](https://dev.2gis.ru)**
2. Нажать **"Получить ключ"**
3. Зарегистрироваться → подтвердить email
4. Ключ будет показан сразу в личном кабинете (вкладка "Проекты")
5. Это API Key для Maps — работает для геокодирования

**Для Catalog API (поиск компаний по рубрике):**
1. В личном кабинете → **"Связаться с нами"** или **support@2gis.ru**
2. Описать сценарий: "B2B SaaS, поиск организаций по рубрике для CRM-обогащения"
3. 2ГИС выдаёт отдельный ключ для Catalog API после одобрения заявки (1–5 рабочих дней)
4. Либо: **[partner.2gis.ru](https://partner.2gis.ru)** → подать заявку как партнёр

**Env var:** `TWOGIS_API_KEY=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`

### 5. Hunter.io — Email Finder

**Зачем:** По домену компании (например, `company.ru`) находит email-адреса сотрудников + паттерн (firstname.lastname@). Даёт confidence score и список источников, где email был найден. Первый шаг в waterfall email-обогащения.

**Бесплатный план:** ✅ **25 поисков/мес** — достаточно для первых пилотных кампаний (25 компаний).

**Платные тарифы:**
| Тариф | Цена | Поисков/мес | Стоимость за поиск |
|-------|------|-------------|-------------------|
| Starter | $49 | 500 | ~$0.10 |
| Growth | $99 | 2,500 | ~$0.04 |
| Business | $199 | 10,000 | ~$0.02 |

**Почему Hunter.io:**
- Наибольшая база найденных email-адресов среди email finders (500M+ профилей)
- Единственный инструмент, который показывает **откуда** найден email (источник)
- Confidence score позволяет отфильтровать ненадёжные адреса (<70%) до отправки
- Person-level email finder: `first_name + last_name + domain` → конкретный email

**Альтернативы:**
| Сервис | Бесплатно | Цена | Точность | Вывод |
|--------|-----------|------|----------|-------|
| **Hunter.io** | 25/мес | $49/500 | ⭐⭐⭐⭐⭐ | Выбор №1 |
| **Snov.io** | 50 кредитов/мес | $39/1K | ⭐⭐⭐⭐ | Хороший fallback |
| Apollo.io | 50 export/мес | $49/1K | ⭐⭐⭐⭐ | Лучше для US/EU рынков |
| RocketReach | 5/мес | $99/мес | ⭐⭐⭐⭐ | Дорого |
| Lusha | 5/мес | $49/мес | ⭐⭐⭐⭐ | Плохое покрытие РФ |
| findthatlead | Нет | $49/мес | ⭐⭐⭐ | Хуже покрытие |

**Как получить (пошагово):**
1. Перейти на **[hunter.io](https://hunter.io)**
2. Нажать **"Sign up free"** (правый верхний угол)
3. Зарегистрироваться: email + пароль (или Google OAuth)
4. Подтвердить email
5. В личном кабинете → **"API"** (левое меню) или **[hunter.io/api-keys](https://hunter.io/api-keys)**
6. API Key уже создан → скопировать

**Env var:** `HUNTER_API_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

### 6. Telegram Bot — Уведомления SDR

**Зачем:** Мгновенные алерты команде продаж: "Лид Иванов из ООО Транслог ответил, хочет звонок завтра — нажмите Позвонить". Inline-кнопки прямо в Telegram. Без бота SDR узнаёт об ответе лида только при следующем входе в CRM.

**Бесплатный план:** ✅ **Полностью бесплатно** — Telegram Bot API не стоит ничего.

**Ограничения:** 30 сообщений/сек общий; 1 сообщение/сек конкретному пользователю. При >10 SDR в команде нужен rate limiter (уже есть в BullMQ).

**Почему Telegram, а не Slack/WhatsApp:**
- Telegram Bot API — публичный, стабильный, без платных тарифов
- В РФ Telegram — основной мессенджер для бизнеса (WhatsApp нестабилен с 2022)
- Slack API требует Slack Workspace + платный тариф для webhook-бота
- WhatsApp Business API: дорого ($0.005–0.08 за сообщение), сложная верификация

**Как получить (пошагово):**
1. В Telegram найти бота **@BotFather** (официальный)
2. Отправить команду `/newbot`
3. BotFather спросит имя бота: `AI Sales OS Alerts` (любое)
4. Затем username: `ai_sales_os_bot` (уникальный, должен заканчиваться на `bot`)
5. BotFather пришлёт токен вида `7123456789:AAE...`
6. Чтобы бот мог писать в личку SDR — каждый SDR должен написать боту `/start` (Telegram требует инициации от пользователя)

> 💡 Для команды: создайте группу, добавьте бота → бот может писать в группу без `/start` от каждого участника.

**Env var:** `TELEGRAM_BOT_TOKEN=7123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

## P2 — Важные, но не блокеры

### 7. Brevo — Резервный email провайдер

**Зачем:** Если Mailgun недоступен или bounced IP попал в блеклист — Brevo автоматически подхватывает отправку. Плюс: бесплатный тир намного щедрее Mailgun.

**Бесплатный план:** ✅ **300 писем/день навсегда** — лучший бесплатный тир среди email провайдеров.

**Платный:** €25/мес за 20,000 писем (~667/день).

**Как получить:**
1. Перейти на **[brevo.com](https://www.brevo.com)**
2. Нажать **"Sign up free"**
3. Заполнить форму → подтвердить email
4. В личном кабинете → **"SMTP & API"** → **"API Keys"** → **"Generate a new API key"**
5. Скопировать ключ

**Env var:** `BREVO_API_KEY=xkeysib-xxxxxxxx...`

## Как добавить ключи в Replit

1. В левом меню Replit нажать на **замок 🔒 "Secrets"**
2. Для каждого ключа: **"New Secret"** → ввести имя (например `OPENAI_API_KEY`) → вставить значение → **"Add Secret"**
3. Все секреты доступны в коде через `process.env.OPENAI_API_KEY` и валидируются через `getEnv()` (`packages/config/src/index.ts`)

> Секреты автоматически доступны во всех воркфлоу без перезапуска. Если изменить — нужно перезапустить воркфлоу вручную.

**Порядок добавления (рекомендуемый):**
```
1. OPENAI_API_KEY          # Без него AI не работает
2. MAILGUN_API_KEY         # Без него письма не уходят
3. MAILGUN_DOMAIN          # sandbox-XXXX.mailgun.org
4. DADATA_API_KEY          # Без него обогащение РФ не работает
5. TWOGIS_API_KEY          # Без него поиск лидов не работает
6. HUNTER_API_KEY          # Без него email waterfall на Pattern fallback
7. TELEGRAM_BOT_TOKEN      # Без него нет уведомлений (некритично для запуска)
8. BREVO_API_KEY           # P2
9. SNOV_API_KEY            # P2
10. ANTHROPIC_API_KEY      # P2
```
