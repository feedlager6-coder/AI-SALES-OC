# User Scenarios — AI Sales OS

## Персонажи

### Алексей — Head of Sales, 35 лет
Руководит командой из 3 SDR в логистическом SaaS стартапе. Продаёт систему управления маршрутами транспортным компаниям. Страдает от: ручного поиска лидов (SDR тратит 60% времени), нет нормального CRM (всё в Excel), письма шаблонные и не открываются.

### Ирина — SDR, 27 лет
Делает 50–80 холодных писем в день. Копирует компании из Google Maps в таблицы, вручную ищет контакты в LinkedIn, пишет почти одинаковые письма. Получает 1–2% ответов. Устала от рутины.

### Дмитрий — CEO/Founder, 42 лет
Хочет знать: сколько денег в пайплайне, откуда приходят лиды, что работает. Сейчас эти данные собираются вручную из Excel-таблиц.

## Сценарий 1: Первый запуск системы (Алексей, новый пользователь)

**Цель**: За 15 минут получить первые квалифицированные лиды и увидеть персонализированное письмо.

```
1. Алексей регистрируется → вводит название компании "RouteAI"
   и описание продукта: "SaaS для оптимизации маршрутов доставки"

2. Onboarding Wizard → Шаг 1: Настройка ICP
   Алексей выбирает:
   • Отрасль: Транспорт и логистика
   • Регион: Москва, Подмосковье  
   • Размер компании: 50–500 сотрудников
   Система показывает: "Найдено 1,247 компаний под ваш ICP"

3. → Шаг 2: Email-аккаунт
   Алексей вводит API-ключ Mailgun (инструкция прямо на экране)

4. → Шаг 3: Демо-поиск
   Система запускает поиск, находит 20 компаний из 2ГИС, обогащает их.
   В реальном времени Алексей видит, как появляются данные:
   "ООО ТрансЛогистик... INN найден... email: ivan@translogistic.ru ✓"

5. → Шаг 4: Первое письмо
   AI генерирует письмо для топового лида:
   
   Тема: "Иван, вопрос про 80 машин ТрансЛогистик"
   
   Тело: "Вижу, что вы открыли 2 вакансии логистов — значит, флот растёт.
   Компании вашего размера обычно теряют 18–25% топлива из-за ручного
   планирования маршрутов. Мы помогли [аналогичная компания] сократить
   это на 23% за первые 3 недели. Актуально?"

   Алексей видит [Оригинально!], нажимает [Отправить]

6. Первый лид отправлен. Алексей получает в Telegram: 
   "🚀 Кампания запущена. Следующие письма отправятся завтра."
```

**Результат**: пользователь получил ценность за < 15 минут.

## Сценарий 2: Ежедневная работа SDR (Ирина)

**Цель**: Начать рабочий день, обработать входящие, выполнить задачи.

```
9:00. Ирина открывает систему → дашборд показывает:
   📥 Входящих: 3 (требуют внимания)
   ✅ Задач на сегодня: 7
   📧 Писем в очереди: 45

9:05. Ирина открывает Inbox:

   [1] ⚡ ХОЧЕТ ЗВОНОК — ООО КурьерМастер
   "Добрый день! Интересно, у нас как раз болит голова с маршрутами.
    Когда можем созвониться?"
   
   AI подсказывает: "Запросил звонок. Рекомендую: предложить время через Calendly"
   
   Ирина нажимает [Отправить Calendly link] — письмо с ссылкой уходит автоматически.
   Система создаёт задачу: "Подтвердить звонок с КурьерМастер"
   Алексей получает уведомление в Telegram: "🔥 КурьерМастер хочет звонок!"

   [2] ❓ ВОПРОС — ЗАО АвтоДоставка
   "А поддерживает ли ваша система импорт заказов из 1С?"
   
   AI предлагает ответ: "Да, есть стандартная интеграция через webhook...
   [Редактировать] [Отправить]"
   
   Ирина быстро проверяет, нажимает [Отправить].

   [3] 💤 НЕ СЕЙЧАС — ИП Смирнов
   "Пока не готовы, у нас Q4 загружен. Напомните в январе"
   
   Система предлагает: "Пауза 75 дней (до 5 января)"
   Ирина нажимает [Подтвердить]. Больше не думает об этом.

9:15. Inbox обработан. Ирина переходит к задачам:
   
   → "Подготовить КП для ООО ТрансЛогистик" (MEETING запланирован)
   Ирина открывает карточку: видит всю историю, боли компании, 
   что обсуждалось в предыдущих письмах.
   
   → "Найти правильный контакт в ГрузТрейд" (wrong_person ответил)
   Ирина видит: "AI предлагает написать директору по логистике или на info@"

10:00. Все входящие обработаны. Ирина возвращается к своей работе.
   Система сама отправит оставшиеся 45 писем в течение дня.
```

**Результат**: SDR тратит < 30 минут на административную работу, остальное — продажи.

## Сценарий 5: Ошибка системы (Graceful Degradation)

**Цель**: Система должна продолжать работу при частичных сбоях.

```
Ситуация: OpenAI API недоступен 30 минут.

1. AI Worker пытается подключиться к OpenAI → timeout
2. Worker автоматически переключается на Anthropic Claude 3.5
3. Письма продолжают генерироваться (с пометкой "generated_by: anthropic")
4. Если оба провайдера недоступны:
   → Письма ставятся в статус PENDING_AI
   → Телеграм алерт: "⚠️ AI недоступен 5+ мин. Генерация писем приостановлена."
   → При восстановлении — автоматически возобновляется

Ситуация: Hunter.io вернул 429 (rate limit)

1. Enrichment Worker получает 429
2. Job retry с exponential backoff: 1мин → 5мин → 15мин → 1час
3. После 4 попыток — пробует Snov.io (fallback провайдер)
4. Лид помечается email_status: "hunter_failed, trying_snov"

Ситуация: Клиент удалил email-аккаунт

1. Email Worker получает authentication error от Mailgun
2. Job помечается как failed
3. Все enrollments этого ящика → PAUSED
4. Уведомление Алексею: "❌ Почтовый ящик spb@routeai.ru отключён.
   [Переподключить]"
```

## Сценарий 6: Добавление новой вертикали (Разработчик)

**Цель**: Подключить вертикаль "Строительство" за 1 день без изменения ядра.

```
Разработчик создаёт:

/packages/verticals/construction/
  icp.yaml         ← правила ICP-скоринга
  prompts.yaml     ← промпты для писем
  sources.yaml     ← приоритет источников данных
  enrichment.yaml  ← специфичные поля обогащения

# icp.yaml
vertical: construction
display_name: "Строительство и девелопмент"
scoring_rules:
  - field: okved_code
    matches: ["41.*", "42.*", "43.*"]
    weight: 30
  - field: employees_count
    range: [50, 2000]
    weight: 20
  - field: open_vacancies_keywords
    contains: ["прораб", "снабженец", "логист"]
    weight: 25
  - field: city
    in: [москва, спб, екатеринбург]
    weight: 15
  - field: has_active_domain
    weight: 10

# prompts.yaml
writer:
  system_addendum: |
    В строительстве ЛПР — генеральный директор или директор по снабжению.
    Боли отрасли: срыв сроков доставки материалов, нет видимости остатков,
    перерасход на перевозку на последней миле.
  first_email_hook: "open_vacancies OR recent_news OR project_announcement"

Система автоматически:
→ Применяет эти правила к новым лидам
→ Использует construction-промпты при генерации писем
→ Показывает вертикаль в фильтрах кампании
```

**Результат**: новая вертикаль за 1 день; ядро не менялось.
