## 1. Лидеры рынка

### Apollo.io — «Всё в одном для SMB»
**Позиция:** Лидер по числу пользователей в сегменте B2B Sales Intelligence + Engagement.  
**Рейтинг:** 4.7/5 на G2 (9,000+ отзывов), 4.5 на Capterra.  
**База данных:** 275M+ контактов, 70M+ компаний.

**Философия:** *"Весь аутбаунд в одном окне"*. Apollo позволяет найти лида, обогатить его данными, запустить email-последовательность и совершить звонок — не выходя из платформы. Для малого и среднего бизнеса это огромная экономия.

**Почему выбирают:**
- Соотношение цена/возможности — лучшее на рынке (есть щедрый Free tier)
- Большая база контактов с прямыми телефонами и email
- Всё в одном: проспектинг + последовательности + диалер + CRM-синк
- Низкий порог входа — первую последовательность можно запустить за 30 минут

**Что делает сильным:**
- Chrome Extension для обогащения прямо с LinkedIn
- AI Writer для персонализации писем
- Встроенный диалер с записью звонков
- Богатые фильтры для поиска (150+ параметров)
- Синхронизация с Salesforce и HubSpot

---

### Clay — «GTM Operating System для data-ops команд»
**Позиция:** Лидер в сегменте обогащения данных и GTM-автоматизации.  
**Рейтинг:** 4.7/5 на G2 (~189 отзывов).  
**Ценообразование (после обновления март 2026):** Free → Launch $185/mo → Growth $495/mo → Enterprise.

**Философия:** *"Соедини любые данные о любом лиде из 150+ источников через единую таблицу"*. Clay — это не CRM и не email-инструмент. Это «spreadsheet с суперсилой» для команд, которые хотят строить собственные GTM-пайплайны.

**Почему выбирают:**
- Waterfall enrichment — если один провайдер не нашёл email, автоматически пробует следующий
- 150+ провайдеров данных в одном инструменте
- Claygent — AI-агент для research по каждому лиду (заменяет ручной Googling)
- Максимальная гибкость — можно собрать любой workflow

**Что делает сильным:**
- Waterfall-подход даёт лучшее покрытие при меньших затратах (~ -50% по стоимости после марта 2026)
- Интеграция с любым email-инструментом (Instantly, Lemlist, Smartlead и др.)
- RevOps-команды строят на Clay целые системы GTM

**Слабость:** Крутая кривая обучения. Non-technical команды недоиспользуют инструмент.

---

### HubSpot CRM — «Платформа для всего»
**Позиция:** Лидер корпоративного рынка, крупнейшая CRM-экосистема.  
**Рейтинг:** Usability 60/100, Support 35/100, Onboarding 40/100.

**Философия:** *"All-in-one: CRM + Marketing + Sales + Service + CMS"*. HubSpot хочет быть единственным инструментом в компании.

**Почему выбирают:**
- Бесплатный старт (Free tier с базовыми CRM-функциями)
- Огромная экосистема интеграций
- Сильный marketing hub
- Большое сообщество и документация

**Что делает сильным:** Глубина. Если команда растёт и усложняется — HubSpot масштабируется вместе с ней.

**Слабость:** Цена растёт экспоненциально. Сложность интерфейса. Многие функции требуют дорогих планов.

---

### Pipedrive — «Визуальный пайплайн»
**Позиция:** Лидер среди малого бизнеса, ориентированный на простоту.  
**Рейтинг:** 4.2/5 на G2.

**Философия:** *"Видишь всё своё pipeline как на ладони"*. Kanban-доска сделок — core UX.

**Почему выбирают:**
- Простота onboarding — за 1 день команда работает
- Визуальный pipeline — интуитивно понятен даже не-техническим командам
- Относительно доступная цена

**Слабость (2026):** Команды перерастают Pipedrive: слабая автоматизация, устаревший AI, слабая аналитика.

---

### Instantly.ai — «Инфраструктура холодного email»
**Позиция:** Лидер в cold email deliverability.  
**Философия:** *"Доставляемость прежде всего"*. Instantly строит инфраструктуру для массового аутбаунда.

**Почему выбирают:**
- Email warmup для неограниченного количества ящиков
- Inbox rotation — автоматически чередует отправителей
- Deliverability-first подход
- Агентский режим — можно обслуживать сотни клиентов

---

### Lemlist — «Персонализация в мультиканале»
**Позиция:** Лидер в персонализированных последовательностях (email + LinkedIn + voice).  
**Философия:** *"Выделись среди спама через настоящую персонализацию"*.

**Почему выбирают:**
- Liquid syntax для динамической персонализации
- Уникальные персонализированные изображения (фото контакта на баннере)
- LinkedIn + Email в единой последовательности
- lemlist database (встроенная база лидов)

---

### Close CRM — «CRM для команд, которые продают звонками»
**Позиция:** Лидер среди inside sales команд с упором на телефонию.  
**Рейтинг:** 4.7/5 на G2, 11,500+ команд, 2B+ звонков/emails/SMS.  
**Ценообразование:** Startup $49/user/mo → Professional $99/user/mo → Enterprise.

**Философия:** *"CRM, который сам продаёт"*. Close — единственная CRM с встроенным AI-агентом (Chloe), который реально совершает звонки, квалифицирует лидов и бронирует встречи.

**Почему выбирают:**
- Лучший built-in диалер на рынке (Power Dialer, Predictive Dialer)
- Chloe AI: автодозвон → транскрипция → обновление CRM → драфт follow-up
- Вся активность (звонки + email + SMS) в одном месте
- Очень быстрый onboarding для inside sales команд

---

### Attio — «Современная CRM нового поколения»
**Позиция:** Восходящий лидер среди tech-стартапов и продуктовых компаний.  
**Рейтинг:** Высокий NPS среди ранних пользователей.

**Философия:** *"CRM, которая работает как современное SaaS, а не как база данных 2009 года"*.

**Почему выбирают:**
- Кастомные объекты — можно моделировать любую бизнес-логику
- Real-time data enrichment
- Современный UX — чистый, быстрый, гибкий
- Нативная автоматизация и workflow builder
- Хорошо интегрируется с email и Calendar

## 2. User Journey Map

### Типичный путь после регистрации (по конкурентам)

```
РЕГИСТРАЦИЯ
    │
    ▼
[Онбординг-опрос]
  "Для чего будете использовать?"
  "Какой у вас размер команды?"
  "Какую CRM используете сейчас?"
    │
    ▼
[Шаг 1: Импорт данных / Подключение почты]
  • Apollo: "Найди 10 лидов прямо сейчас"
  • HubSpot: "Подключи Gmail/Outlook"
  • Pipedrive: "Создай первую сделку"
  • Close: "Подключи телефон / Создай первый лид"
  • Attio: "Импортируй CSV или подключи Gmail"
    │
    ▼
[Шаг 2: Первое действие с реальными данными]
  • Apollo: Поиск лидов по фильтрам → добавить в список
  • Clay: Создать таблицу → добавить лидов → выбрать провайдер обогащения
  • Instantly: Подключить ящики → настроить warmup → создать кампанию
  • Close: Создать лид → позвонить из браузера
    │
    ▼
[Первая ценность (Time-to-Value)]
  • Apollo: ~15 минут (нашёл email, добавил в sequence)
  • Clay: ~2-4 часа (требует настройки workflow)
  • Instantly: ~30 минут (отправил первое письмо)
  • Close: ~5-10 минут (позвонил из CRM)
  • HubSpot: ~1-2 часа (сложный setup)
  • Pipedrive: ~20-30 минут (создал pipeline + сделки)
  • Lemlist: ~45 минут (создал первую последовательность с персонализацией)
  • Attio: ~30 минут (настроил кастомный объект, импортировал данные)
```

**Вывод:** Лучшие продукты достигают First Value за 15-30 минут. Clay — исключение (его аудитория принимает сложность).

**Количество кликов до первого результата:**
- Лучшие: Close (3-4 клика до первого звонка), Instantly (4-5 до первого письма)
- Средние: Apollo (6-8), Pipedrive (5-6)
- Сложные: HubSpot (10+), Clay (15+)

## 5. Что раздражает пользователей

### Системные проблемы, повторяющиеся у большинства конкурентов:

**Apollo.io:**
- Кредитная система — непрозрачная и непредсказуемая (главная жалоба 2025-2026)
- Качество данных: устаревшие email (bounce rate), дубли
- Deliverability: массовые рассылки попадают в спам
- Перегруженный интерфейс — слишком много фич

**Clay:**
- Крутая кривая обучения — требует технических знаний
- Dual credit system (Data Credits + Actions) — сложно планировать бюджет
- Без data-ops специалиста команда застрянет

**HubSpot:**
- Цена: растёт экспоненциально при масштабировании
- Bloatware: 80% функций большинство не использует
- Медленный и тяжёлый интерфейс
- Onboarding требует консультанта или дни setup
- Support очень низкий по оценкам (35/100)

**Pipedrive:**
- Слабая автоматизация (жалобы 2025-2026)
- Медленное развитие продукта
- Слабая аналитика
- Нет нативного Email sequences (нужны интеграции)
- AI-функции запоздали

**Close CRM:**
- Реальная цена выше, чем рекламируемая (диалер + телефонные минуты = ~$129/user вместо $49)
- Менее гибкий data model
- Не лучший для сложных B2B продаж с длинным циклом

**Instantly:**
- Нет настоящего CRM — только outreach
- Аналитика ограничена
- Управление несколькими командами сложно

**Lemlist:**
- LinkedIn автоматизация в серой зоне (риск бана)
- Персонализированные изображения технически сложны в setup
- Цена высокая для малых команд

**Attio:**
- Слишком дорого для стартапов (Free plan очень ограничен)
- Отсутствие built-in email sequences
- Нет встроенного диалера

### Общие жалобы на весь рынок:
1. **Data entry overhead** — руками вводить всё после каждого звонка
2. **Разрыв между инструментами** — нужен Clay + Instantly + Lemlist + отдельная CRM
3. **AI-функции — маркетинг, а не ценность** — большинство AI-фич не используются
4. **Цена непредсказуема** — кредиты, лимиты, тиры создают uncertainty
5. **Нет единой картины взаимодействий** — звонки в одном месте, email — в другом

## 6. UX-паттерны

### Что есть у всех (стандарт индустрии):

**Карточка компании:**
```
┌─────────────────────────────────────────────┐
│ [Логотип]  Название компании                │
│            Отрасль • Размер • Локация       │
│            Сайт • LinkedIn                  │
├─────────────────────────────────────────────┤
│ КОНТАКТЫ (2)     СДЕЛКИ (1)    ЗАДАЧИ (3)  │
├─────────────────────────────────────────────┤
│ TIMELINE / ACTIVITY FEED                    │
│  ✉ Email отправлен — 2ч назад              │
│  📞 Звонок 3 мин — вчера                   │
│  📝 Заметка — 3 дня назад                  │
└─────────────────────────────────────────────┘
```

**Карточка контакта:**
```
┌─────────────────────────────────────────────┐
│ [Аватар]  Имя Фамилия                       │
│           Должность • Компания              │
│           📧 email  📞 телефон  🔗 LinkedIn │
├─────────────────────────────────────────────┤
│ [Написать] [Позвонить] [Задача] [Заметка]  │
├─────────────────────────────────────────────┤
│ TIMELINE                                    │
│  Все активности в хронологии               │
└─────────────────────────────────────────────┘
```

**Pipeline (Kanban):**
```
│ Новый    │ Квалиф.  │ Предложен. │ Закрыт    │
│ ($240K)  │ ($180K)  │ ($95K)     │ ($430K)   │
├──────────┼──────────┼────────────┼───────────┤
│ [Сделка1]│[Сделка3] │ [Сделка5]  │[Сделка7]  │
│ [Сделка2]│[Сделка4] │            │           │
```

**Почему эти паттерны повторяются у всех:**
- Менеджер по продажам думает в терминах: компания → контакт → сделка → активность
- Timeline отвечает на главный вопрос: "Что у нас происходит с этим лидом?"
- Kanban даёт мгновенный ответ: "Где моя воронка?"
- Quick actions (Написать / Позвонить / Задача) снижают friction при работе

## 7. Рабочий день менеджера по продажам

### Типичный день SDR (Outbound):

```
08:00 — Утро: приоритизация
  → Открыть CRM
  → Посмотреть Task Queue на сегодня
  → Проверить ответы на email (из вчерашних рассылок)
  → Проверить пропущенные звонки
  → AI приоритизирует: "Кому звонить первым?"

09:00-12:00 — Блок звонков
  → Открыть лида из списка
  → Быстрый AI-brief: "Что мы знаем об этой компании?"
  → Позвонить → Не дозвонился
     → Выбрать статус: "Не отвечает"
     → Создать задачу: "Перезвонить завтра в 10:00"
     → Отправить voicemail / email-follow-up (1 клик)
  → Дозвонился → разговор
     → AI транскрибирует
     → Выбрать итог: "Заинтересован / Не интересует / Перезвонить"
     → Быстрая заметка
     → AI предлагает follow-up письмо → одобрить / отредактировать

12:00-13:00 — Обед

13:00-15:00 — Email-работа
  → Ответить на входящие
  → Запустить новую последовательность для нового списка лидов
  → Персонализировать топ-10 приоритетных контактов

15:00-17:00 — Pipeline-работа
  → Обновить статусы сделок
  → Добавить заметки по встречам
  → Подготовить предложения
  → Запланировать follow-up задачи

17:00-17:30 — Закрытие дня
  → Проверить метрики дня (звонки, письма, ответы)
  → Завтра: AI предлагает план на завтра
```

**Как это реализовано у конкурентов:**

| Этап | Close | Apollo | HubSpot | Pipedrive |
|------|-------|--------|---------|-----------|

## 8. Работа со звонками

### Стандарт рынка 2026:

**Базовый минимум (есть у всех нормальных CRM):**
- Click-to-call прямо из карточки лида
- Запись разговора
- Лог звонка (дата, длительность, направление)
- Статус звонка (дозвонился / не ответил / занято)
- Комментарий после звонка

**Средний уровень (Close, HubSpot Sales Hub):**
- Power Dialer — автоматически набирает следующий номер
- Voicemail drop — записанное сообщение одним кликом
- Транскрипция звонка (AI)
- Call recording с AI-саммари
- Аналитика по звонкам (количество, конверсия)

**Топовый уровень (Close Chloe, 2026):**
- AI-агент сам совершает звонки и квалифицирует лидов
- Real-time coaching во время звонка
- Автообновление CRM после звонка
- Автодрафт follow-up письма на основе транскрипции
- Анализ тональности разговора

### Как можно реализовать лучше в AI Sales OS:

**Проблема у конкурентов:** После звонка менеджер должен вручную заполнить:
- Что обсудили (заметка)
- Итог звонка (статус)
- Следующий шаг (задача)
- Follow-up письмо

Это занимает 3-7 минут на каждый звонок. При 50 звонках в день = 2.5-6 часов на администрирование.

**Предложение для AI Sales OS:**
```
Звонок завершён → AI автоматически:
  1. Транскрибирует разговор
  2. Определяет итог: заинтересован / нет / перезвонить
  3. Создаёт заметку из ключевых моментов
  4. Предлагает статус сделки и следующий шаг
  5. Генерирует follow-up письмо
  
Менеджер только: [Одобрить] или [Редактировать] — 10-15 секунд.
```

**Workflow звонка:**
```
[Открыть карточку лида]
  → AI brief: "Последний контакт 3 дня назад. Обсуждали..."
  → [Позвонить] одна кнопка
  → Набор → Разговор (запись + транскрипция в реальном времени)
  → Завершить звонок
  → [Панель итога появляется автоматически]
      AI предлагает: Статус / Заметка / Задача / Follow-up email
  → Менеджер подтверждает за 15 секунд
  → Следующий лид загружается автоматически
```

### Статусы звонков (стандарт):
- 📞 Дозвонился — разговор состоялся
- 📵 Не отвечает — нет ответа
- 🔔 Занято — перезвонить
- 📬 Voicemail — оставил сообщение
- ❌ Неверный номер
- 🚫 Отказ от звонков

## 10. Аналитика

### Что реально используют ежедневно:

**Менеджер (индивидуально):**
- Сколько звонков сделал сегодня/на этой неделе

### Что почти никогда не открывают:
- Сложные custom dashboards с 20+ метриками
- Cohort analysis
- Revenue attribution by channel
- Geographic heatmaps
- Competitive intelligence dashboards

### Лучшая аналитика в 2026:
**Close:** Простая, быстрая, ориентированная на action.
**HubSpot:** Самая мощная, но требует настройки.

**Вывод:** Не нужно строить BI-платформу в MVP. Нужно 5-7 метрик, которые помогают менеджеру принять решение прямо сейчас.

## 12. Что можно сделать лучше конкурентов

### 1. Unified Workspace (главная идея)

Сейчас sales-стек среднего SDR:
- Clay (обогащение)
- Instantly или Lemlist (outreach)
- Apollo (поиск контактов)
- HubSpot или Pipedrive (CRM)
- Close или отдельный диалер (звонки)
- Slack или Telegram (коммуникация с командой)

Это 5-6 инструментов. AI Sales OS может стать **единым рабочим пространством**, объединив:
- Prospecting (поиск + обогащение через плагин-систему)
- Outreach (email последовательности + LinkedIn + звонки)
- CRM (компании, контакты, pipeline)
- AI (не отдельная фича, а сквозной слой через всё)
- Communications Hub (вся история в одном месте)

### 2. Zero Data Entry

**Проблема:** Менеджеры тратят 40-60% времени в CRM на data entry.

**Решение:** AI автоматически:
- После звонка → создаёт заметку из транскрипции
- После email → обновляет статус лида
- После встречи → создаёт summary + задачи
- При добавлении компании → автоматически обогащает из открытых источников

### 3. Intent-based pipeline (не Kanban, а Activity Queue)

**Проблема с Kanban:** Показывает состояние, но не говорит что делать дальше.

**Предложение:** Параллельный режим "Today's Queue" — AI определяет приоритет:
```
ОЧЕРЕДЬ НА СЕГОДНЯ (приоритет AI):
1. 🔥 Technopark LLC — открыли письмо 2 раза вчера. Позвонить сейчас
2. ⏰ DigitalWave — обещал ответить "до пятницы". Напомнить
3. 📧 MetroGroup — не отвечает 5 дней. Попробовать другой канал
4. 🆕 DataSoft — новый входящий лид. Квалифицировать
```

### 4. Multichannel из коробки

Сейчас 2026: продажи идут через email + звонок + LinkedIn + Telegram/WhatsApp.

AI Sales OS должен поддерживать все каналы в единой последовательности:
```
Шаг 1: Email (день 1)
Шаг 2: Звонок (день 3)
Шаг 3: Voicemail если нет ответа (день 3)
Шаг 4: LinkedIn connection request (день 5)
Шаг 5: LinkedIn message (день 7)
Шаг 6: Telegram/WhatsApp (если есть номер, день 10)
```

### 5. Plugin Architecture (уже в проекте)

Это настоящее конкурентное преимущество. Clay показал, что **"платформа + провайдеры"** — мощная модель. AI Sales OS уже имеет plugin system. Это нужно развивать:
- Каждый источник данных (2GIS, HH.ru, Dadata, Hunter) — плагин
- Каждый канал коммуникации (email, звонок, Telegram) — плагин
- Waterfall enrichment: если один плагин не нашёл → пробует следующий

## 18. Лучшие UX-практики рынка

### 1. Progressive Disclosure для AI
Не показывать все AI-возможности сразу. Показывать в контексте:
- На карточке лида: "Хотите AI-саммари по этой компании?"
- После звонка: "Создать заметку автоматически?"
- При написании письма: "AI может персонализировать под этот лид"

### 2. Zero-click Data Entry
Любое действие должно приводить к автоматическому обновлению данных:
- Отправил email → статус лида обновился автоматически
- Звонок завершён → открылась панель итога с предзаполненными полями
- Встреча в Calendar → создался лид / обновился статус

### 3. Context-aware Actions
Каждый экран должен знать "что пользователь хочет сделать дальше":
- Открыл карточку лида утром → предлагает: Позвонить / Написать / Просмотреть историю
- Звонок не отвечает → предлагает: Оставить voicemail / Создать задачу / Написать email

### 4. Command Palette (как в Notion, Linear)
`Cmd/Ctrl + K` для быстрых действий:
- "Создать задачу для ..."
- "Найти компанию ..."
- "Позвонить в ..."
- "Запустить sequence ..."

### 5. Keyboard-first для опытных пользователей
Менеджер, совершающий 50 звонков в день, должен делать всё без мыши:
- Tab для перехода между полями
- Enter для подтверждения
- Shortcuts для статусов звонков

### 6. Mobile: Только для входящих уведомлений
В MVP не нужен полноценный mobile. Нужно:
- Уведомление: "Лид ответил на письмо"
- Быстрый ответ прямо из уведомления
- Просмотр карточки перед звонком с мобильного

### 7. Inbox Zero для задач
Концепция "рабочего дня": с утра видишь список задач → закрываешь по одному → к вечеру inbox пуст. Gamification через completion, а не badges.

## 19. Предложения по архитектуре AI Sales OS

### Что в текущей архитектуре правильно:
- ✅ **Plugin-система** — это ключевое конкурентное преимущество. Развивать, не трогать.
- ✅ **Event-driven через BullMQ** — правильный подход для async AI-обработки
- ✅ **Multi-tenant** — необходимо для SaaS
- ✅ **Monorepo (Turbo + pnpm)** — удобно для команды
- ✅ **Drizzle ORM** — хороший выбор, type-safe
- ✅ **Company как основная сущность** — правильно (в B2B продажах продают компании, не людям)

### Что стоит пересмотреть / добавить:

**1. Event Sourcing для Timeline**
Timeline — центральный UX. Все действия (звонки, письма, смены статуса) должны пишутся как **events**, не мутировать состояние. Это обеспечит:
- Полную историю изменений
- Возможность "проиграть" историю заново
- AI-анализ паттернов продаж

**2. Channel Abstraction Layer**
Сейчас в архитектуре email + Mailgun. Нужен абстрактный слой `Channel`:
```typescript
interface Channel {
  send(message: Message): Promise<void>;
  receive(): AsyncIterator<Message>;
  getHistory(contactId: string): Promise<Message[]>;
}
// Implementations: EmailChannel, TelegramChannel, WhatsAppChannel, CallChannel
```

**3. AI Layer как сквозной сервис**
AI не должен быть отдельным модулем — он должен быть доступен везде:
- После события → AI обрабатывает и предлагает action
- При создании письма → AI помогает
- При просмотре лида → AI делает brief

```
packages/ai/ → ai-service (уже есть, хорошо)
  → email-composer
  → call-summarizer  
  → lead-researcher (Claygent-аналог)
  → next-action-suggester
```

**4. Enrichment Queue с Waterfall**
```
Лид добавлен → EnrichmentJob создаётся
→ Plugin 1 (Dadata) пробует по ИНН → нашёл → стоп
→ Plugin 2 (Hunter) пробует email → нашёл → стоп
→ Plugin 3 (2GIS) добавляет геолокацию
→ Лид обогащён → UI обновляется в реальном времени (WebSocket)
```

**5. Real-time через WebSocket**
Менеджер должен видеть:
- "Лид открыл письмо" → уведомление в реальном времени
- После AI-обогащения → карточка обновилась без перезагрузки
- Новый входящий → всплывающее уведомление

## 22. Рекомендации до подключения реальных API

### Критически важно сделать до подключения провайдеров:

**1. Правильная архитектура хранения API-ключей**
- Шифрование API-ключей провайдеров (уже есть `ENCRYPTION_KEY` в проекте) ✅
- Ключи хранятся per-tenant, не глобально ✅
- Rotation ключей без downtime

**2. Rate limiting и retry logic**
Каждый плагин должен иметь:
- Exponential backoff при ошибках провайдера
- Queue с приоритетами для обогащения
- Graceful degradation: если плагин упал → пробуем следующий

**3. Credit / cost tracking**
Каждый вызов к внешнему API стоит денег. Нужно:
- Логировать стоимость каждого обогащения
- Показывать статистику расходов в admin panel
- Устанавливать лимиты per-tenant

**4. Dry-run mode (заглушки без реальных API)**
Для demo и тестирования нужны mock-ответы всех плагинов. Это позволит:
- Показывать продукт без реального API-ключа
- Тестировать workflows
- Обучать новых пользователей

**5. UX-первичность над данными**
Не ждать, пока все плагины будут подключены. Запустить с:
- Ручным вводом данных
- 1-2 ключевыми плагинами (Dadata для RU)
- Заглушками для остальных

**6. Обогащение как фоновый процесс**
Пользователь добавляет компанию → видит её сразу с базовыми данными → обогащение происходит в фоне → карточка обновляется. Никогда не блокировать UI ради обогащения.

## 23. Если бы строили AI Sales OS с нуля сегодня

### Что оставить из текущей архитектуры:
- ✅ **Plugin architecture** — это главное конкурентное преимущество, правильное решение
- ✅ **Event-driven с BullMQ** — правильно для async AI
- ✅ **Company как центральная сущность** — правильная модель для B2B
- ✅ **Multi-tenant** — необходимость для SaaS
- ✅ **Monorepo** — правильно для небольшой команды
- ✅ **TypeScript end-to-end** — type safety критична в сложной системе

### Что спроектировал бы иначе:

**1. Event Sourcing с первого дня**
Вместо мутирующих таблиц — `events` как источник правды. Timeline получается бесплатно. AI может анализировать паттерны. Откат изменений возможен. Это архитектурное решение сложно добавить потом.

**2. Channel abstraction с первого дня**
Email — лишь один из каналов. Telegram, WhatsApp, звонки — равноправные каналы. Если не заложить абстракцию сразу, потом дорого рефакторить.

**3. AI как сквозной слой, не модуль**
Вместо `packages/ai` как отдельного модуля — AI как middleware, доступный в любой точке системы через единый интерфейс.

**4. Offline-first / Optimistic UI**
CRM используется при плохом интернете (менеджер в дороге). Оптимистичные обновления критичны для UX.

**5. Меньший scope в MVP**
Сосредоточиться на одном killer use case: "Позвонил → AI заполнил CRM → следующий звонок". Всё остальное — потом.

**6. Pricing с первого дня**
Не бесплатный tier как у Apollo (9000 пользователей на Free = дорого для инфраструктуры). С первого дня: trial 14 дней → платная подписка. Это важно для product-market fit поиска.
