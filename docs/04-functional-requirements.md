# Functional Requirements — AI Sales OS

## FR-01. Управление лидами и компаниями

### FR-01.1 Создание и хранение
- Система должна создавать записи Lead, Contact, Company из любого источника: ручной ввод, импорт CSV, API, парсинг, webhook
- Каждая Company имеет: ИНН (уникальный), название, сайт, телефоны, email-домен, категория ОКВЭД, город, численность, выручка (если доступно), стек технологий
- Каждый Contact имеет: ФИО, должность, email (с verification статусом), телефон, LinkedIn URL, связь с Company
- Дедупликация: при создании проверяется совпадение по ИНН → домену → email → (название + город)

### FR-01.2 ICP-скоринг
- Система рассчитывает ICP Score (0–100) для каждого лида на основе настраиваемых весовых правил
- Правила задаются через UI без кода (размер компании, отрасль, регион, наличие вакансий, технологии)
- Score пересчитывается автоматически при обновлении данных лида

### FR-01.3 Статусы и пайплайн
- Лид проходит этапы: `new → enriching → qualified → contacted → replied → meeting_scheduled → closed_won → closed_lost`
- Переходы между этапами — по ручному действию или автоматическим триггерам
- История изменений статуса хранится с временными метками и причиной

## FR-02. Lead Generation (поиск новых лидов)

### FR-02.1 Источники поиска
- **2ГИС**: поиск компаний по категории + городу; результат → список компаний с базовыми данными
- **HH.ru**: поиск работодателей по отрасли + размеру; вакансии как сигнал роста и болей
- **ЕГРЮЛ**: обогащение по ИНН — официальное название, дата регистрации, статус, директор
- **VK**: публичные страницы компаний по ключевым словам
- **Telegram**: публичные каналы и группы (только публичные)
- **Ручной импорт**: CSV/XLSX с маппингом полей

### FR-02.2 Настройка поиска
- Пользователь задаёт ICP-параметры через UI: отрасль, регион, размер, ключевые слова
- Система формирует поисковые запросы к источникам автоматически
- Поиск запускается по расписанию (cron) или вручную

## FR-03. Lead Enrichment (обогащение данных)

### FR-03.1 Водопадное обогащение (Waterfall)
- Порядок провайдеров настраивается; следующий вызывается только если предыдущий не вернул нужное поле
- Провайдеры email: Hunter.io → Snov.io → Apollo.io → собственный поиск по паттернам
- Email верификация: синтаксис → MX-запись → SMTP-проверка (catch-all detection)
- Обогащение телефона: Dadata.ru → ручная база

### FR-03.2 AI-обогащение
- Агент автоматически изучает сайт компании и извлекает: боли, продукты, клиентов, технологии
- Агент проверяет последние новости компании (Google News API)
- Агент анализирует вакансии на HH.ru: что ищут → какие боли растут

### FR-03.3 Результаты обогащения
- Каждое поле имеет источник (провайдер) и confidence (0–1)
- Низкий confidence → флаг для ручной проверки
- Обогащение не перезаписывает ручные данные без явного разрешения

## FR-04. Email Outreach

### FR-04.1 Последовательности (Sequences)
- Создание многошаговых последовательностей: шаг = email/пауза/условие
- Условные ветки: если открыл → шаг A; если не открыл → шаг B
- Максимум 10 шагов на последовательность
- Персонализация через переменные + AI-генерация тела письма

### FR-04.2 Прогрев отправителя
- Автоматическое прогревание новых email-ящиков через seed pool
- Мониторинг репутации домена (DMARC, SPF, DKIM валидация)
- Алерты при попадании в спам

### FR-04.3 Аналитика email
- Метрики: sent, delivered, bounced (hard/soft), opened, clicked, replied, unsubscribed, complained
- Tracking pixel + redirect-link трекинг
- A/B тест: рандомное распределение вариантов с статистической значимостью

### FR-04.4 Compliance
- Автоматическое добавление ссылки отписки
- Соблюдение временных ограничений (не отправлять в нерабочее время)
- Глобальный список unsubscribed через все кампании
- Поддержка 152-ФЗ (хранение персданных на серверах в РФ опционально)

## FR-05. AI Layer

### FR-05.1 Генерация писем
- На входе: данные лида (компания, роль, боли, новости, вакансии)
- На выходе: персонализированный текст письма (тема + тело), 3 варианта
- Модель: GPT-4o или Claude 3.5 Sonnet (настраивается)
- Промпт-шаблоны по отраслям (транспорт, строительство, IT и т.д.)
- Возможность редактировать результат перед отправкой

### FR-05.2 Классификация ответов
- Входящий reply автоматически классифицируется: `interested` / `not_now` / `not_interested` / `wrong_person` / `request_call` / `out_of_office` / `unsubscribe`
- Для `interested` и `request_call` — уведомление SDR + авто-создание задачи
- Для `out_of_office` — пауза последовательности и возобновление через N дней

### FR-05.3 Рекомендации
- После каждого ответа агент предлагает следующий шаг с обоснованием
- Агент анализирует паттерны успешных сделок и рекомендует наиболее похожие лиды

## FR-06. CRM

### FR-06.1 Сущности
- **Company** (organization): базовый объект
- **Contact**: физическое лицо, связано с Company (M:M)
- **Deal**: потенциальная сделка, привязана к Company и Contact
- **Activity**: любое взаимодействие (email, звонок, встреча, заметка)
- **Task**: задача для SDR (с дедлайном и приоритетом)
- **Sequence**: email-последовательность
- **Campaign**: набор Sequence + список лидов

### FR-06.2 Кастомизация
- Кастомные поля для Company/Contact/Deal без кода
- Кастомные этапы пайплайна для разных вертикалей
- Теги и сегменты на любую сущность

### FR-06.3 Timeline / Activity Feed
- Каждая сущность имеет хронологическую ленту всех взаимодействий
- Фильтрация по типу активности
- Поиск по всей базе (full-text)

---

## FR-07. Аналитика и дашборд

### FR-07.1 Операционный дашборд (SDR)
- Задачи на сегодня
- Статус активных последовательностей
- Последние ответы, требующие внимания
- Новые лиды в очереди

### FR-07.2 Управленческий дашборд
- Воронка продаж (конверсия по этапам)
- Метрики email кампаний (open, reply, meeting rate)
- Revenue pipeline (прогноз выручки)
- Топ-10 источников лидов по конверсии

### FR-07.3 Экспорт
- Любые данные экспортируются в CSV/XLSX
- Отчёты доступны через API

---

## FR-08. Мультитенантность и доступ

- Workspace = изолированная среда для одной компании-клиента
- Роли: Owner, Admin, Manager, SDR (с разными правами)
- Приглашение пользователей по email
- SSO через Google (опционально)
- Логирование всех действий (audit log)
