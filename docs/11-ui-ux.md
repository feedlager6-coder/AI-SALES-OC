# UI/UX Design — AI Sales OS

## Дизайн-принципы

1. **Intent → Result**: каждый сценарий начинается с намерения пользователя и заканчивается результатом; всё между ними — автоматика
2. **Zero Learning Curve**: пользователь без опыта в продажах понимает что делать без туториалов
3. **Действие на расстоянии одного клика**: пользователь не должен проходить 3 экрана чтобы написать первым
4. **AI предлагает — человек решает**: всегда видно что сделал AI и почему; можно отредактировать; критичные действия требуют подтверждения
5. **Минимум экранов**: если задачу можно решить на одном — она решается на одном
6. **Пустой экран — это провал**: новый пользователь видит поле «Кого вы ищете?», а не пустую очередь задач
7. **Системный язык — только в коде**: слова ICP, Hunt, Pipeline, SDR, Waterfall, Sequences — только в API и документации; в UI — пользовательский язык

|-----------|-------|------------|

| Forms | React Hook Form + Zod | Type-safe, shared schemas с backend |

| Date picker | react-day-picker | Locale-aware |
| Command menu | cmdk | ⌘K поиск |
| Notifications | Sonner | Toast уведомления |
| Real-time | Socket.IO client | WebSocket обновления |

## Мобильный опыт

MVP — **desktop first**, но ключевые действия работают на мобильном:
- Telegram Bot заменяет мобильный дашборд для SDR "в поле"
- Веб-версия адаптивна (responsive) но не оптимизирована
- Mobile-first — v2

## Состояния загрузки и ошибок

| Ситуация | UX-решение |
|---------|-----------|
| Загрузка таблицы | Skeleton placeholders (не спиннер) |
| AI генерирует письмо | Streaming текст (как ChatGPT) |
| Обогащение в процессе | Прогресс-бар + промежуточные данные |
| Ошибка API | Toast с кратким описанием + кнопка Retry |
| Нет лидов | Empty state с иллюстрацией и кнопкой "Найти лиды" |
| Offline | Banner "Нет подключения, данные могут быть устаревшими" |

## Onboarding Flow (первый вход)

```
1. Регистрация (30 секунд)
   → только email + пароль; никаких вопросов о команде и инструментах
   ↓
2. Экран «Кого вы ищете?»
   → одно поле; пример запроса подсказывает формат
   ↓
3. Interactive Intent
   → система показывает как поняла запрос; максимум 1 уточняющий вопрос
   ↓
4. Результаты (через ~2 минуты)
   → список компаний с контактами и подсказками почему они интересны
   ↓
5. Первое сообщение
   → AI подготовил черновик; пользователь читает → [Отправить] или [Изменить]
   → Готово. Первый потенциальный клиент — в работе.
```

**Что здесь никогда не происходит:**
- Нет «настройте ICP»
- Нет «подключите источник данных»
- Нет «у вас пока нет задач»
- Нет «импортируйте CSV для начала»

**Цель onboarding**: пользователь находит релевантного клиента и отправляет первое персонализированное сообщение за < 10 минут.

## Accessibility

- WCAG 2.1 AA соответствие
- Keyboard navigation полная (Tab, Enter, Escape, Arrow keys)
- ARIA labels на всех интерактивных элементах
- Contrast ratio: минимум 4.5:1 для текста
- Focus indicators видимые
- Screen reader совместимость (тестируется NVDA + Chrome)
