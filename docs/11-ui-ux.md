# UI/UX Design — AI Sales OS

## Дизайн-принципы

1. **Информация > Декор**: интерфейс показывает данные, а не украшения
2. **Действие на расстоянии одного клика**: SDR не должен проходить 3 экрана чтобы отправить письмо
3. **AI-прозрачность**: всегда видно, что сделал AI и почему; можно отредактировать
4. **Progressive disclosure**: сложные опции скрыты, базовый флоу — простой
5. **Плотность данных**: дашбород как Bloomberg Terminal (не как лендинг)

## Технологии

| Компонент | Выбор | Обоснование |
|-----------|-------|------------|
| Framework | Next.js 15 App Router | SSR, RSC, встроенный routing |
| UI Components | shadcn/ui | Kopируется в проект, нет vendor lock-in |
| Styling | Tailwind CSS 4 | Utility-first, тёмная тема из коробки |
| Icons | Lucide React | Consistent, tree-shakeable |
| Tables | TanStack Table v8 | Виртуализация, sorting, filtering |
| Charts | Recharts | React-native, достаточно для дашборда |
| Forms | React Hook Form + Zod | Type-safe, shared schemas с backend |
| Drag & Drop | @dnd-kit | Kanban, reordering |
| Date picker | react-day-picker | Locale-aware |
| Command menu | cmdk | ⌘K поиск |
| Notifications | Sonner | Toast уведомления |
| Real-time | Socket.IO client | WebSocket обновления |

## Мобильный опыт

MVP — **desktop first**, но ключевые действия работают на мобильном:
- Telegram Bot заменяет мобильный дашборд для SDR "в поле"
- Веб-версия адаптивна (responsive) но не оптимизирована
- Mobile-first — v2

## Состояния загрузки и ошибок

| Ситуация | UX-решение |
|---------|-----------|
| Загрузка таблицы | Skeleton placeholders (не спиннер) |
| AI генерирует письмо | Streaming текст (как ChatGPT) |
| Обогащение в процессе | Прогресс-бар + промежуточные данные |
| Ошибка API | Toast с кратким описанием + кнопка Retry |
| Нет лидов | Empty state с иллюстрацией и кнопкой "Найти лиды" |
| Offline | Banner "Нет подключения, данные могут быть устаревшими" |

## Onboarding Flow (первый вход)

```
1. Создание workspace (название компании, сайт, описание продукта)
   ↓
2. Подключение email-ящика для отправки (Mailgun / SMTP)
   ↓
3. Настройка ICP (какие компании ищем — отрасль, регион, размер)
   ↓
4. Первый поиск лидов (демо-режим: 20 лидов из 2ГИС бесплатно)
   ↓
5. AI обогащает первые 5 лидов (видно в реальном времени)
   ↓
6. Создание первой кампании (guided wizard)
   ↓
7. Отправка первого письма → 🎉
```

**Цель onboarding**: пользователь получает ценность (первые лиды + персонализированное письмо) за < 15 минут.

## Accessibility

- WCAG 2.1 AA соответствие
- Keyboard navigation полная (Tab, Enter, Escape, Arrow keys)
- ARIA labels на всех интерактивных элементах
- Contrast ratio: минимум 4.5:1 для текста
- Focus indicators видимые
- Screen reader совместимость (тестируется NVDA + Chrome)
